# Glossary

- **Adapter**: a boundary implementation that accesses another bounded context through its public contract.
- **Bounded Context**: a domain boundary with its own model, invariants, and data ownership.
- **Break-glass**: exceptional, time-limited access for life-safety response with strong authentication and review.
- **Calling Journey**: a communal discernment process; it is not a declaration of divine certainty.
- **Domain Event**: a versioned fact emitted after a domain change.
- **Field**: a researched region, community, network, or diaspora setting; not a target score for conversion.
- **Local leader priority**: the requirement that credible local people progressively own interpretation, decisions, resources, and succession.
- **Mission OS**: the bounded context coordinating the mission lifecycle.
- **Outbox**: transactionally stored events published asynchronously and idempotently.
- **RLS**: PostgreSQL row-level security enforcing tenant and role isolation in the database.
- **Safeguarding**: prevention, reporting, escalation, and accountable response for harm and vulnerability.
- **Tenant**: an organization-scoped security and data boundary.
