# Roadmap

Batch 0 establishes charter, boundaries, feature flags, outbox, audit and lineage, safeguarding, and release gates. Batch 1 adds tenant, permission, privacy, consent, retention, minor protection, and strong-session controls. Batches 2 through 10 deliver field intelligence, calling, preparation, training, sending, compliance, deployment, content access, member care, local leadership, and authority transfer. Batch 11 completes operations, evaluation, model governance, observability, E2E security, release, rollback, and disaster recovery.

Dependencies are strictly forward: `B0 -> B1 -> B2 -> B3 -> B4 -> B5 -> B6 -> B7 -> B8 -> B9 -> B10 -> B11`. A batch cannot open while its prior gate contains a P0 or P1 failure.
