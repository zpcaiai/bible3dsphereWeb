# Definition of Done

A Mission OS skill is done only when its domain model, reversible migration, repository and service, typed API, server-enforced authorization, accessible responsive UI, audit policy, observability, documentation, and rollback plan are complete.

It includes domain, repository, API permission, tenant-isolation, UI component, critical E2E, and high-risk negative tests. Loading, empty, error, forbidden, and redacted states are implemented. Internationalized strings are used. AI work records consent, prompt and model versions, structured output, latency and usage metadata, editable human review, and rejection.

The repository's lint, formatting check, type check, unit tests, integration tests, migration smoke test, security tests, and relevant E2E paths must pass. A skipped security assertion, disabled RLS, fabricated success, swallowed exception, or client-only authorization makes the skill incomplete.
